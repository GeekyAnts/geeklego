## Accessibility

- The `as` prop controls the semantic heading level in the document outline — screen readers announce it
- Heading levels must never be skipped (no h2 → h4 jump — the `size` prop is what to use for visual adjustments)
- One `<h1>` per page — use `as="h2"` and below for all section headings
- Use `id` on the heading and `aria-labelledby` on the containing section to create accessible landmark regions
- No keyboard interaction — Heading is a display-only component
