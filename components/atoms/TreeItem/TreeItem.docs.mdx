## Accessibility

- Renders as `<li role="treeitem">` — must be placed inside a `<ul role="tree">` container
- `aria-expanded` reflects expanded state on nodes with children
- `aria-selected` reflects whether the node is the active selection
- `aria-level`, `aria-setsize`, and `aria-posinset` communicate tree position to assistive technologies
- `aria-disabled` set when `isDisabled` is true
- Expand toggle has a contextual `aria-label` ("Expand [label]" / "Collapse [label]") and is `aria-hidden` when the node has no children
- All decorative icons carry `aria-hidden="true"`
- Supports roving tabindex — exactly one item holds `tabIndex={0}` at any time; the parent `TreeView` manages this
- `focus-visible` focus ring on the selection button for keyboard navigation

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus into and out of the tree (roving tabindex) |
| ArrowDown | Moves focus to the next visible tree node |
| ArrowUp | Moves focus to the previous visible tree node |
| ArrowRight | Expands a collapsed node; moves to first child if already expanded |
| ArrowLeft | Collapses an expanded node; moves to parent if already collapsed |
| Enter | Selects the focused node |
| Space | Selects the focused node |
| Home | Moves focus to the first node in the tree |
| End | Moves focus to the last visible node in the tree |
