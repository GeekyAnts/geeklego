## Accessibility

- Root: `<div>` container
- Tab list: `<div role="tablist">` with `aria-label` (resolved from `i18nStrings.listLabel`, default `"Tabs"`) and `aria-orientation`
- Tab trigger: `<button role="tab">` with `aria-selected` and `aria-controls` pointing to its panel
- Panel: `<div role="tabpanel">` with `aria-labelledby` pointing to its tab, and `tabIndex={0}` so the panel is focusable
- Disabled tabs receive `aria-disabled="true"` and are skipped during arrow-key navigation
- Icons inside tab triggers are wrapped with `aria-hidden="true"`
- Loading skeleton carries `aria-busy="true"` via `getLoadingProps`
- Roving tabindex pattern: exactly one tab has `tabIndex={0}` at a time; all others are `-1`

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus to the tab list as a single unit, then into the active panel |
| ArrowRight / ArrowLeft | Move focus between tabs (horizontal orientation); selection follows focus |
| ArrowDown / ArrowUp | Move focus between tabs (vertical orientation); selection follows focus |
| Home | Move focus to the first non-disabled tab |
| End | Move focus to the last non-disabled tab |
