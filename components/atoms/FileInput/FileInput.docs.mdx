## Accessibility

- Wraps a native `<input type="file">` — fully keyboard accessible without ARIA scaffolding
- Always provide an accessible label via `aria-label` or an external `<label htmlFor>`
- Error state sets `aria-invalid="true"` automatically; link to the error message with `aria-describedby`
- Loading state sets `aria-busy="true"` and disables the input
- For multiple file selection, describe multi-select in the label (e.g. "Upload images, multiple allowed")
- `accept` is a browser hint only — always validate file types server-side as well

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus to the file input |
| Enter / Space | Opens the system file picker |
