## Accessibility

- Trigger receives `aria-describedby` pointing to the tooltip panel via `cloneElement` — screen readers read it as supplementary description
- Panel has `role="tooltip"` — announced as a tooltip by screen readers
- Panel is always in the DOM (never removed), so the `aria-describedby` reference is always valid
- `aria-hidden` toggles on the panel to hide it from assistive technology when not visible
- Focus-triggered tooltips appear immediately regardless of `delayMs`
- `disabled={true}` completely suppresses the tooltip — use when the trigger already has a sufficient visible label
- `i18nStrings.panelLabel` sets an `aria-label` on the panel for rich or non-textual content

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus to the trigger element |
| Escape | Dismisses the tooltip |
