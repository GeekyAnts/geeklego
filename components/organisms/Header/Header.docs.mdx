## Accessibility

- Semantic `<header>` element — announced as the "banner" landmark by screen readers
- `<nav aria-label="Primary">` (or the value from `i18nStrings.navLabel`) — announced as "Primary navigation"
- Mobile toggle button uses `aria-expanded` and `aria-controls` paired with the mobile panel `id` via `useId()`
- Mobile toggle label changes between "Open menu" and "Close menu" (overridable via `i18nStrings`)
- `NavItem` with `isActive` renders `aria-current="page"` on its link
- `Header.Brand` link `href` is always passed through `sanitizeHref()` to strip dangerous protocols
- Escape key dismisses the mobile navigation panel via `useEscapeDismiss`
- Clicking outside the header also dismisses the mobile panel via `useClickOutside`
- All decorative icons include `aria-hidden="true"`
- Duplicate `<nav>` landmarks (desktop + mobile panel) each have an `aria-label` to distinguish them
- Icon-only action buttons must provide a text child which becomes the `aria-label`

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus through brand link, nav items, and action buttons |
| Enter / Space | Activates the focused link or button |
| Escape | Dismisses the mobile navigation panel when open |
