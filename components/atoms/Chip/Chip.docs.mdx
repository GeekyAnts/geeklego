## Accessibility

- Interactive chips render as `<button aria-pressed>` — screen readers announce toggle state
- `interactive=false` chips render as `<span>` — use for removable tags where the × button handles interaction
- Never set `interactive=true` with `onRemove` — nesting `<button>` inside `<button>` is invalid HTML
- `aria-disabled="true"` set when `disabled` is true
- Leading icon is `aria-hidden="true"` — the label carries full meaning
- Wrap a group of filter chips in a container with `role="group"` and `aria-label`
- Wrap removable tags in `role="list"` / `role="listitem"` for screen reader enumeration

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus to the chip |
| Enter / Space | Toggles selected state (interactive mode) |
| Enter | Activates remove button (when focused) |
