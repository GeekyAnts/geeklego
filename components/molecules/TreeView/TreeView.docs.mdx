### TreeNode

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique stable identifier. Used as React key. Required. |
| `label` | `string` | Display label. Truncated with ellipsis on overflow. Required. |
| `icon` | `ReactNode` | Optional leading icon — lucide-react icons recommended. |
| `children` | `TreeNode[]` | Child nodes. Presence of this array (even empty) determines whether the node has an expand toggle. |
| `disabled` | `boolean` | Prevents interaction with this node. |
| `loading` | `boolean` | Shows a spinner on this node while async children are loading. |
| `badge` | `string \| number` | Trailing badge count or label. |

## Accessibility

- Rendered as `<ul role="tree">` — announced as a tree widget by screen readers
- `aria-label` or `aria-labelledby` is required to give the tree an accessible name
- Roving `tabindex` pattern — exactly one node has `tabIndex={0}` at a time; all others are `-1`
- Selected node has `aria-selected="true"`
- Disabled nodes have `aria-disabled="true"` and cannot receive focus
- Loading nodes render a spinner — the node row carries accessibility hints for the loading state
- `multiSelect={true}` sets `aria-multiselectable="true"` on the tree root
- Expand toggle button has `aria-expanded` and `aria-controls` pointing to the subtree

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus into the tree (resumes at the last focused node) |
| Arrow Up / Arrow Down | Moves focus between visible nodes |
| Arrow Right | Expands a collapsed node; moves focus to the first child if already expanded |
| Arrow Left | Collapses an expanded node; moves focus to the parent if already collapsed |
| Enter / Space | Selects the focused node |
| Home | Moves focus to the first node |
| End | Moves focus to the last visible node |
