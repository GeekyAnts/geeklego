## Collapsible Variants

The `collapsible` prop controls collapse behaviour. `"icon"` collapses to a narrow icon rail, `"offcanvas"` slides the panel off-screen, and `"none"` keeps it always visible.

## Sub-Menu

`Sidebar.MenuSub` nests a second-level list beneath a parent `Sidebar.MenuItem`. The active sub-item uses `isActive` on `Sidebar.MenuSubButton`. Use `ChevronRight` as a `suffix` on the parent button to indicate expandability.

## Loading

## Active, Disabled, Badge, and Action

## Accessibility

- `<Sidebar>` renders as `<aside>` — a complementary landmark announced as "Sidebar navigation" by screen readers
- `<Sidebar.Content>` is the scrollable nav wrapper
- `<Sidebar.Footer>` is scoped inside the `<aside>` landmark
- `Sidebar.MenuButton` with `isActive` sets `aria-current="page"` on the active item
- `Sidebar.MenuButton` with `disabled` sets `aria-disabled="true"` and `tabIndex={-1}`
- `Sidebar.GroupAction` and `Sidebar.MenuAction` require `aria-label` — they are icon-only buttons
- In icon-rail mode (`collapsible="icon"` + `state="collapsed"`), `Sidebar.MenuButton` hides label text and exposes `tooltip` as the native `title` attribute for screen reader announcement
- `Sidebar.Rail` receives `tabIndex={-1}` and is reachable by pointer only
- The global keyboard shortcut (default Cmd+B / Ctrl+B) toggles the sidebar from anywhere on the page; it is configurable via the `shortcut` prop on `SidebarProvider`

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus into the sidebar; first stop is `Sidebar.Trigger` |
| Tab (again) | Enters `Sidebar.Content`; first nav item receives focus |
| ArrowDown / ArrowUp | Moves between `Sidebar.MenuButton` items (roving tabindex) |
| Enter / Space | Activates a link or triggers a button action |
| Tab (from last nav item) | Exits the nav and reaches the footer `Sidebar.MenuButton` |
| Cmd+B / Ctrl+B | Toggles sidebar open/closed from anywhere on the page |
