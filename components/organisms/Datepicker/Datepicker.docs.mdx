## Accessibility

- Input uses `aria-haspopup="dialog"`, `aria-expanded`, and `aria-controls` to link to the calendar panel
- Calendar button has `aria-label` from i18n strings
- Calendar panel is `role="dialog"` with `aria-modal="true"` and `aria-labelledby` pointing to the month heading
- Calendar grid uses `role="grid"` with `th scope="col" abbr` for weekday headers
- Each day cell is a button with `aria-selected`, `aria-current="date"` for today, and `aria-disabled` for out-of-range days
- Each day button has a full-text `aria-label` (e.g. "15 March 2026")
- Month/year heading has `aria-live="polite"` to announce month changes
- Error messages use `role="alert"` for immediate screen reader announcement
- Focus is trapped within the calendar panel when open
- Focus returns to the input when the calendar closes

### Keyboard Interaction

| Key | Context | Action |
|-----|---------|--------|
| ArrowDown | Input focused | Opens calendar |
| Enter / Space | Day focused | Selects date, closes calendar |
| ArrowLeft / ArrowRight | Grid focused | Previous / next day |
| ArrowUp / ArrowDown | Grid focused | Previous / next week |
| Home / End | Grid focused | Start / end of week |
| PageUp / PageDown | Grid focused | Previous / next month |
| Shift+PageUp / Shift+PageDown | Grid focused | Previous / next year |
| Escape | Calendar open | Closes calendar, returns focus to input |
| Tab | Calendar open | Cycles within focus trap |
