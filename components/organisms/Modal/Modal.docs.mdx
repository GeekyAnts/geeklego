## Accessibility

- Rendered with `role="dialog"` and `aria-modal="true"` to announce as a modal dialog to screen readers
- `aria-labelledby` references the title heading when a `title` prop is provided
- `aria-label` fallback used when no `title` is provided, resolved via `i18nStrings.dialogLabel`
- `aria-busy` set on the panel when `loading={true}`
- Focus is trapped inside the dialog using `useFocusTrap` — Tab and Shift+Tab cycle only within modal content
- Focus returns to the triggering element on close
- Escape key dismisses the modal via `useEscapeDismiss`
- Close button carries an explicit `aria-label` resolved via `i18nStrings.closeLabel`
- The backdrop close button icon is wrapped with `aria-hidden="true"`
- Renders into a portal on `document.body` to avoid stacking context and overflow clipping issues

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus to the next focusable element inside the modal |
| Shift + Tab | Moves focus to the previous focusable element inside the modal |
| Escape | Closes the modal |
