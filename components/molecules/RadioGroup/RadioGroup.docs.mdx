## Accessibility

- Semantic `<fieldset>` element with `<legend>` — provides correct group semantics natively (implicit `role="group"`)
- No explicit `role` needed — `<fieldset>` carries it natively
- `aria-required` set on the `<fieldset>` when `required` is true; SR-only text appended to the legend
- `aria-describedby` wired to the hint or error paragraph id
- Error paragraph uses `role="alert"` — announced live by screen readers when it appears
- `disabled` attribute on `<fieldset>` disables all child inputs natively
- Individual options accept `disabled` for per-option disable without affecting the group
- Decorative `*` required marker has `aria-hidden="true"`; accessible text is SR-only

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus into the radio group (to the first or selected option) |
| Arrow Down / Arrow Right | Moves to the next radio option and selects it |
| Arrow Up / Arrow Left | Moves to the previous radio option and selects it |
| Tab (out) | Moves focus out of the group to the next focusable element |
