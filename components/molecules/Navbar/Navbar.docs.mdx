### NavbarItemDef

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique stable identifier. Used as React key. Required. |
| `href` | `string` | Link destination. Required. |
| `label` | `string` | Visible link text. Required. |
| `icon` | `ReactNode` | Optional leading icon. |
| `isActive` | `boolean` | Marks this item as the current page. |
| `disabled` | `boolean` | Non-interactive, visually muted. |
| `badge` | `string \| number` | Optional badge count shown on the trailing edge. |

## Accessibility

- Rendered as `<nav>` with `aria-label` — announced as a navigation landmark
- Items are `<a>` elements inside `<ul>/<li>` — communicate list structure to screen readers
- Active item receives `aria-current="page"` via the underlying NavItem atom
- Disabled items are non-interactive and announced as dimmed
- Multiple `<nav>` elements on the same page must each have a unique `aria-label`
- Schema.org `SiteNavigationElement` Microdata available via the `schema` prop

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus between navigation links |
| Enter | Navigates to the focused link |
