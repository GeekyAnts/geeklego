## Accessibility

- `role="progressbar"` on the container div — no native circular progress element exists
- `aria-valuenow` set to the rounded percentage in determinate state; omitted when indeterminate
- `aria-valuemin` is always `0`; `aria-valuemax` is always `100` in determinate state
- `aria-valuetext` provides a human-readable percentage string (e.g. `"65%"`)
- `aria-label` from the `label` prop — always provide a meaningful label for standalone usage
- `aria-busy="true"` set in indeterminate state to signal active loading to assistive technology
- `aria-disabled="true"` set when the `disabled` prop is true
- The inner `<svg>` carries `aria-hidden="true"` and `focusable="false"` — screen readers narrate the outer container only
- The component is not keyboard-focusable — it is a read-only status widget

### Keyboard Interaction

This component has no keyboard interaction. It is a display-only progress indicator. Focus management is the responsibility of the surrounding page or dialog that contains it.
