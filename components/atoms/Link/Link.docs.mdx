## Accessibility

- Renders as a native `<a>` element — full browser and assistive technology support
- `external` automatically adds `target="_blank"` with `rel="noopener noreferrer"` for security; consider adding `aria-label="Link text (opens in a new tab)"` for screen readers
- `disabled` removes the link from the tab order and adds `aria-disabled="true"` — it does not use the HTML `disabled` attribute (anchors don't support it)
- Always use descriptive link text — avoid "click here", "read more", or "here" as standalone labels
- `focus-visible` focus ring for keyboard navigation

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus to the link |
| Enter | Activates the link |
