## Accessibility

- `<div>` container with two `<button>` stepper controls flanking a native `<input type="number">`
- Decrement button `aria-label` defaults to "Decrease value" (i18n-configurable via `i18nStrings.decrementLabel`)
- Increment button `aria-label` defaults to "Increase value" (i18n-configurable via `i18nStrings.incrementLabel`)
- Both buttons include `aria-controls` pointing to the inner input's id
- Boundary buttons get `disabled` and `aria-disabled="true"` when value is at `min` or `max`
- Error state sets `aria-invalid="true"` on the inner input; pair with `aria-describedby` for the error message

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus to decrement button → input → increment button |
| Enter / Space | Activates the focused stepper button |
| Arrow Up / Down | Browser-native increment/decrement on the focused input |
