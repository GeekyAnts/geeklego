### StepItem

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique stable identifier. Used as React key. Required. |
| `label` | `string` | Step label shown below (horizontal) or beside (vertical) the indicator. Required. |
| `description` | `string` | Optional supporting text rendered under the label. |
| `status` | `'error'` | Per-step error override — marks the step as errored regardless of `activeStep`. |

## Accessibility

- Rendered as `<ol aria-label="Steps">` — screen readers announce it as an ordered list
- Active step indicator receives `aria-current="step"`
- Completed steps include a screen-reader-only suffix (e.g. "(Completed)") via `VisuallyHidden`
- Error steps include a screen-reader-only suffix (e.g. "(Error)") via `VisuallyHidden`
- When `onStepClick` is provided, completed and active indicators render as `<button>` elements — keyboard-focusable and activatable
- Connector lines are `aria-hidden="true"` — decorative only
- `i18nStrings.listLabel` overrides the `<ol>` aria-label for contextual descriptions (e.g. "Checkout steps")

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus between clickable step buttons (when `onStepClick` is provided) |
| Enter | Activates the focused step button |
| Space | Activates the focused step button |
