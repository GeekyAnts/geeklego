## Accessibility

- Each trigger is a `<button type="button">` wrapped in an HTML heading (configurable via `headingLevel`, default `h3`)
- Panel has `role="region"` and `aria-labelledby` pointing to its trigger id
- `aria-expanded` on each trigger reflects the open/closed state
- `aria-controls` links each trigger to its panel by id
- Disabled items use `aria-disabled` and `disabled` attributes together
- Collapsed panels receive `inert` to prevent Tab focus into hidden content
- A screen-reader-only hint ("Expand" / "Collapse") supplements `aria-expanded` for older assistive technology
- Panel expand/collapse uses CSS grid transition, which respects `prefers-reduced-motion` automatically

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Move focus between accordion triggers and other focusable elements |
| Shift + Tab | Move focus in reverse |
| Enter / Space | Expand or collapse the focused trigger's panel |
