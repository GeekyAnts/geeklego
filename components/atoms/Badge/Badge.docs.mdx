## Accessibility

- Text badge content is announced by screen readers inline with surrounding content
- `variant="dot"` renders no visible text — `dotLabel` is required to provide an accessible name via `aria-label`
- Add `role="status"` to dot badges that represent live or dynamic states (e.g. online/offline)
- Numeric count badges should include `aria-label` when used without surrounding context (e.g. `aria-label="12 unread messages"`)
- Color alone is never the sole indicator — variant shape and label together convey state
- No keyboard interaction — Badge is a display-only component
