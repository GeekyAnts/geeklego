## Accessibility

- `<label>` is linked to the input via `htmlFor`/`id` — `id` is auto-generated with `useId()` when omitted
- Error message uses `role="alert"` — announced immediately by screen readers when it appears
- `aria-describedby` links the input to the hint text (when no error) or the error message (when present)
- The label mirrors the disabled state visually
- `required` renders a visual indicator on the label and sets the native `required` attribute on the input

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus to the date input |
| Arrow keys | Adjusts the focused date component (day, month, year) — browser-native behaviour |
| Up / Down | Increments or decrements the focused component |
