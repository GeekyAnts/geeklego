## Accessibility

- The icon wrapper has `aria-hidden="true"` — purely decorative, not announced by screen readers
- The `title` renders as a `<p>` — no heading level is assumed; use surrounding heading hierarchy as needed
- For dynamically-inserted empty states (e.g. after a search returns no results), add `role="status"` so assistive technology announces the change
- The `action` slot receives focus naturally via its own interactive element — no extra ARIA needed
- No keyboard interaction specific to the component — focus moves through interactive children normally
