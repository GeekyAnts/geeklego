## Accessibility

- Renders as a group of `<button>` elements — each button has `aria-pressed` for selection state
- `aria-label` on the component is required — describes the group purpose (e.g. "View mode")
- Icon-only segments require `aria-label` on the individual segment option object
- Uses roving tabindex — Tab enters the group and focuses the selected segment; Arrow keys navigate within
- Disabled segments have `aria-disabled="true"` and are skipped by the roving tabindex
- Group `disabled` adds `aria-disabled` to the container and all segments

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Enters the group (focuses selected segment) |
| Arrow Left / Right | Moves focus and selection between segments |
| Home | Jumps to the first segment |
| End | Jumps to the last segment |
| Space / Enter | Selects the focused segment |
