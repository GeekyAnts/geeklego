## Accessibility

- Renders as a native `<label>` element — associates with an input via `htmlFor` + matching `id`
- The red asterisk for `required` is `aria-hidden="true"` — screen readers hear `"(required)"` via a `sr-only` span instead
- The `optional` text is rendered as visible text and read directly by screen readers
- `hasError` changes label color only — `aria-invalid` belongs on the associated input element
- No keyboard interaction — Label is a display-only element that improves input click/touch targets
