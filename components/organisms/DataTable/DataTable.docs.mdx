## Sortable

Sorting is fully controlled. Pass `sortState` and `onSortChange` to enable sort toggling on columns marked `sortable: true`. The consumer is responsible for sorting the data array. Clicking a sortable column cycles: none, ascending, descending, none.

## Row Selection

Pass `selectedKeys` (a Set of row key strings) and `onSelectionChange` to enable row selection. A checkbox column is prepended automatically. The header checkbox supports select-all and indeterminate state.

## Accessibility

- Wrapped in a `role="region"` container with `aria-label` (from caption or explicit aria-label prop)
- `aria-busy` set on the region during loading state
- Native `<table>` element with `<caption class="sr-only">` for screen reader identification
- `<th scope="col">` on all column headers so screen readers announce the column on every cell
- `aria-sort` on sortable `<th>` elements communicates current sort state (ascending, descending, or none)
- Sort button `aria-label` describes the next action: "Sort [column] ascending", "Sort [column] descending", "Clear sort for [column]"
- `aria-selected` on each `<tr>` communicates row selection state
- Header checkbox shows indeterminate state when a subset of rows is selected
- Skeleton rows are `aria-hidden="true"` to prevent screen reader noise during loading
- All sort buttons and checkboxes meet the 24x24 px minimum touch target (WCAG 2.5.8)

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus to sort buttons and row checkboxes in document order |
| Enter | Activates a focused sort button or toggles a checkbox |
| Space | Activates a focused sort button or toggles a checkbox |
