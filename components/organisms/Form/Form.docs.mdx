## Accessibility

- Semantic `<form noValidate>` — native form element. `noValidate` disables browser validation popups so custom errors render in `Form.Field`.
- Labels via `Label` atom — explicit `<label htmlFor>` association (WCAG 1.3.1).
- Errors use `role="alert"` — announced immediately on first appearance (WCAG 4.1.3).
- Required fields: `aria-required="true"` on the control + asterisk + sr-only "(required)" on label.
- Loading: `aria-busy="true"` on the `<form>` element when `loading` is true.
- `Form.Field` generates deterministic IDs from `htmlFor`: hint element gets `[htmlFor]-hint`, error element gets `[htmlFor]-error`. Pass these in `aria-describedby` on the control for full WCAG 1.3.1 compliance.
- Use `<Fieldset disabled>` to propagate disabled state to all descendant controls natively — do not thread individual `disabled` props.

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Move focus to the next form control |
| Shift+Tab | Move focus to the previous form control |
| Enter | Submit the form (when focused on a text input or submit button) |
| Space | Check or uncheck checkboxes; activate buttons |
| Arrow keys | Navigate radio groups; increment sliders |
