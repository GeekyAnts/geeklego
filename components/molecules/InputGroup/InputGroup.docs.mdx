## Accessibility

- `<div role="group">` wraps the addon elements and inner `<input>` as one logical unit
- `aria-label` on the group is recommended when no visible `<label>` is present; pass it directly to `InputGroup`
- `aria-disabled`, `aria-invalid`, and `aria-busy` are managed by the inner Input atom and set on the `<input>` element
- Pass `aria-describedby` via `...rest` to connect an external hint or error message to the inner `<input>`
- Decorative addon content (icons, short text labels) should have `aria-hidden="true"` applied directly to the content node
- Semantic addon content (e.g., country code `+1` that helps users understand what to type) should remain visible to screen readers — omit `aria-hidden` and rely on the group `aria-label` to provide context
- Interactive suffix elements (e.g., a `Button`) retain their own accessible name and keyboard behaviour

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus into the inner input |
| Standard input keys | Character input, selection, deletion |
| Tab (with interactive suffix) | Continues focus to the Button or other interactive element in the suffix slot |
