## Accessibility

- Renders as a custom ARIA combobox (`role="combobox"`) with a listbox dropdown (`role="listbox"`) and individual options (`role="option"`) — not a native `<select>`
- The trigger is a `<button>` element; keyboard navigation is implemented manually via the component
- When `label` is provided, it is linked via `htmlFor`/`id` — provides the accessible name for the trigger button
- Without `label`, pass `aria-label` directly to the component
- `aria-expanded` reflects the open/closed state of the dropdown on the trigger button
- `aria-controls` links the trigger to the listbox panel
- `aria-activedescendant` tracks the currently focused option within the open listbox
- `error` sets `aria-invalid="true"` on the trigger — pair with `aria-describedby` pointing to an error message
- `disabled` sets both HTML `disabled` and `aria-disabled` on the trigger button

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus to the select |
| Space / Enter | Opens the dropdown |
| Arrow Up / Down | Navigates between options |
| Escape | Closes the dropdown |
