## Accessibility

- Non-interactive cards: add `role="article"` for self-contained content regions
- Interactive cards: `tabIndex={0}` is auto-applied; add `role="button"` and `aria-label` when the whole card is a navigation target
- Loading state: use `aria-busy="true"` and a descriptive `aria-label` on the card root; skeleton content should be `aria-hidden`
- Action buttons in `Card.Header` should have descriptive `aria-label` values when their label alone is ambiguous
- Focus ring is applied via `focus-visible:focus-ring` on interactive cards

### Keyboard Interaction

| Key | Action |
|-----|--------|
| Tab | Moves focus to an interactive card or controls inside |
| Enter | Activates an interactive card |
| Space | Activates an interactive card |
