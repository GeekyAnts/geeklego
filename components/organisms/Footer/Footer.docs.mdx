## Accessibility

- Semantic `<footer>` element — implicit `contentinfo` landmark role when a direct child of `<body>`
- `aria-label` on `<footer>` set from i18n `footerLabel` (default: `"Footer"`)
- Each `Footer.Nav` column renders a `<nav>` landmark with `aria-label` from the `navAriaLabel` prop or i18n `navLabel`
- When rendering multiple `Footer.Nav` columns, always provide a unique `navAriaLabel` to each one — otherwise all nav landmarks share the same accessible name, making them indistinguishable in landmark navigation
- `Footer.Nav` renders a heading (`h3` by default) for each column — override `headingLevel` when the page structure requires a different level
- All `href` values in `Footer.Brand` are passed through `sanitizeHref()` automatically, stripping `javascript:`, `vbscript:`, and `data:text/html` protocols
- External links (via `Link` with `external={true}`) receive `rel="noopener noreferrer"` and a visually-hidden "(opens in new tab)" cue automatically
